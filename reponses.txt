a) La commande "git fetch" récupère les dernières modifications du dépôt distant et les enregistre localement, mais ne met pas à jour le code local. En revanche, la commande "git pull" récupère également les dernières modifications du dépôt distant, mais met également à jour le code local en fusionnant les modifications dans la branche active. En d'autres termes, "git pull" équivaut à un "git fetch" suivi d'un "git merge".
b) Pour annuler les modifications locales non validées dans Git, vous pouvez utiliser la commande "git checkout -- ." pour annuler toutes les modifications locales dans le répertoire de travail ou "git checkout -- <nomdufichier>" pour annuler les modifications d'un fichier spécifique. Cela permet de restaurer le fichier dans l'état où il se trouvait au dernier commit.

c) Pour supprimer une branche distante dans Git, vous pouvez utiliser la commande "git push <remote> --delete <nombranche>" pour supprimer la branche à distance. Par exemple, si la branche que vous souhaitez supprimer est "develop" et le dépôt distant est "origin", vous pouvez exécuter la commande "git push origin --delete develop".

d) Pour voir l'historique des modifications pour un fichier spécifique dans Git, vous pouvez utiliser la commande "git log <nomdufichier>". Cela affiche l'historique des modifications pour ce fichier, avec les détails tels que les messages de commit, les auteurs, les dates et les identifiants de commit. Vous pouvez également utiliser des options supplémentaires pour personnaliser la sortie de la commande, telles que "--pretty=format:'%h %ad | %s%d [%an]'" pour afficher les informations de manière plus concise.
